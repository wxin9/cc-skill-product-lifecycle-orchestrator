# 产品需求文档

## 产品愿景

这是一个任务管理工具，帮助团队高效协作和管理任务。

## 核心功能

### F01 — 用户登录

用户能够通过邮箱和密码登录系统。

### F02 — 任务评论功能 [新增]

团队成员可以在任务下留言讨论，支持以下特性：
- 评论内容支持富文本格式
- 支持回复他人评论（嵌套回复，最多 3 层）
- 评论支持 @提及其他团队成员
- 评论历史记录可追溯
- 评论支持编辑和删除（仅限评论作者或管理员）
- 支持表情符号和图片粘贴（GIF、PNG、JPG，单张图片 ≤ 5MB）

**用户场景**：
- 团队成员对任务细节有疑问时，可在任务下留言讨论
- 项目经理可在任务下补充说明或反馈
- 开发人员可在任务下记录技术决策或问题

### F03 — 附件上传功能 [新增]

支持在任务中上传附件，具体要求：
- 支持常见文件格式（PDF、Word、Excel、图片、压缩包等）
- 单个文件大小限制：50MB
- 每个任务附件数量限制：20个
- 支持拖拽上传和点击选择文件
- 上传进度可视化
- 支持附件预览（图片、PDF）
- 支持附件下载
- 附件版本管理（上传同名文件时提示覆盖或保留历史版本）
- 附件权限控制：仅任务成员可下载附件

**用户场景**：
- 设计师上传设计稿
- 开发人员上传技术文档
- 测试人员上传测试报告或截图
- 客户上传需求文档或参考资料

## 非功能需求

### 性能要求

- **响应时间**：
  - API 接口响应时间 < 200ms（P95）
  - 页面加载时间 < 1.5s（首次加载）
  - 数据库查询响应时间 < 50ms
- **并发能力**：
  - 支持 100 并发用户
  - API 接口吞吐量 ≥ 500 QPS
- **存储容量**：
  - 附件存储：支持至少 100GB 总容量
  - 单个文件大小限制：50MB
  - 每个任务附件数量限制：20个
- **实时性**：
  - 评论实时性：支持 WebSocket 实时推送新评论
  - 消息推送延迟 < 500ms

### 安全需求

- **身份认证**：
  - 用户密码必须使用 bcrypt 或 Argon2 加密存储
  - 实施登录失败次数限制（5 次失败后锁定账户 15 分钟）
  - Session 过期时间为 30 分钟，支持自动续期
  - 敏感操作（如修改密码、删除任务）需要二次验证
- **数据传输**：
  - 所有 API 接口必须使用 HTTPS 加密传输
  - 实施 CORS 策略，仅允许指定域名访问
- **输入验证**：
  - 所有用户输入必须进行 XSS 和 SQL 注入防护
  - 文件上传必须进行类型和大小验证
  - 评论内容过滤敏感词和恶意代码
- **权限控制**：
  - 附件权限控制：仅任务成员可下载附件
  - 评论编辑删除权限：仅评论作者或管理员可操作
- **审计与监控**：
  - 记录所有关键操作日志（登录、文件上传、任务修改）
  - 定期安全审计和依赖包漏洞扫描
  - 异常登录行为检测和告警

### 可用性要求

- 系统可用性 ≥ 99.5%
- 数据备份：每日自动备份，保留 30 天
- 故障恢复时间 < 1 小时

## 技术约束

- **附件存储方案**：使用对象存储服务（AWS S3 或阿里云 OSS）
- **评论数据存储**：需要设计评论表结构，支持嵌套查询和分页
- **实时通信**：使用 WebSocket 实现评论实时推送
- **前端框架**：建议使用 React 或 Vue.js
- **后端框架**：建议使用 Node.js (Express) 或 Python (FastAPI)
- **数据库**：建议使用 PostgreSQL 或 MySQL
