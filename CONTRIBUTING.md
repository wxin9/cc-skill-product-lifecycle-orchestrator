# Contributing to Product-Lifecycle Skill

感谢您对 Product-Lifecycle Skill 项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请在 GitHub Issues 中提出：

1. 检查是否已有相关 Issue
2. 提供清晰的问题描述
3. 包含复现步骤（如果是 bug）
4. 说明您的期望行为

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 遵循 Python PEP 8 代码风格
- 保持代码简洁和可读性
- 添加适当的注释
- 确保所有测试通过

## 开发环境设置

```bash
# 克隆仓库
git clone https://github.com/wxin9/cc-skill-product-lifecycle.git
cd cc-skill-product-lifecycle

# 创建虚拟环境（可选）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

## 提交信息规范

请使用清晰的提交信息，格式如下：

```
<类型>: <简短描述>

<详细描述>
```

类型可以是：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

## 许可证

通过贡献代码，您同意您的贡献将根据本项目的 Apache 2.0 许可证进行许可。

# 贡献指南（英文）

Thank you for your interest in the Product-Lifecycle Skill project! We welcome all forms of contributions.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature suggestion, please file it in GitHub Issues:

1. Check if there's already a related Issue
2. Provide a clear description of the problem
3. Include reproduction steps (if it's a bug)
4. Explain your expected behavior

### Submitting Code

1. Fork this repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

### Code Style

- Follow Python PEP 8 code style
- Keep code clean and readable
- Add appropriate comments
- Ensure all tests pass

## Development Setup

# Clone the repository

cd cc-skill-product-lifecycle

python -m venv venv
source venv/bin/activate  # Linux/Mac

## Commit Message Guidelines

Please use clear commit messages with the following format:

```
<type>: <short description>

<detailed description>
```

Types can be:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation update
- `style`: Code formatting
- `refactor`: Refactoring
- `test`: Testing related
- `chore`: Build/tooling related
